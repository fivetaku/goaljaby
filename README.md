# goaljaby (골잡이)

<p align="center">
  <img src="assets/goaljaby-hero-01.png" alt="goaljaby" width="320">
</p>

> **A bridge skill that turns a PRD folder into a runnable `/goal` with built-in verify and recover loops.**

goaljaby takes a PRD folder (manual or generated by `/show-me-the-prd`) and auto-produces **five documents wrapping a verify/recover loop** — VALIDATION, RECOVERY, PLAN, PROGRESS, and a `/goal` command — then hands off to Claude Code (paste-back) or Codex CLI (handoff prompt) after a human approval gate.

[Quick Start](#quick-start) • [Why goaljaby?](#why-goaljaby) • [How it works](#how-it-works) • [Outputs](#outputs) • [Task types](#task-types) • [Requirements](#requirements)

---

## Quick Start

### 1. Install

```
/plugin marketplace add https://github.com/fivetaku/gptaku_plugins.git
/plugin install goaljaby
```

Restart Claude Code.

### 2. Invoke

With an existing PRD directory:

```
/goaljaby ./PRD/
```

Without a PRD (goaljaby will offer to delegate to `/show-me-the-prd`):

```
/goaljaby
```

Or natural language:

- "골잡이 호출"
- "골 셋업 만들어줘"
- "PRD에서 골 만들어줘"
- "set up goal from PRD"

### 3. The flow

goaljaby runs ten steps in order. The most important ones:

| Step | What it does |
|------|--------------|
| **0** | PRD pre-check. No PRD → delegate to `/show-me-the-prd`, or accept a single-line goal in light mode |
| **2** | Pick target CLI (Claude Code / Codex / both) |
| **3** | Task type + validation methods + strictness (1–2 interview rounds) |
| **4** | Confirm milestones (≤5) |
| **5** | Slot-fill 5 documents from templates |
| **6** | Auto-compact `goal-command.md` to ≤4,000 chars (5-stage priority) |
| **7** | Deterministic verification (`grep -P` against 5 PROTECTED_CLAUSES + char count) |
| **9** | **Human approval gate** (mandatory — cannot be bypassed) |
| **10** | Paste-back guide for Claude Code, or handoff prompt for Codex |

## Why goaljaby?

- **A PRD alone isn't enough** — A PRD says *what* to build. `/goal` requires *how to prove it's done* and *how to recover when it goes wrong*. Missing either, the goal stops at "looks plausible" or drifts in the wrong direction.
- **Verify + Recover is the real workload** — VALIDATION (acceptance mapping + Not Done If) and RECOVERY (3-attempt rule + scope lock) are what keep a multi-turn goal honest. goaljaby auto-generates both from the PRD.
- **4,000-char compact is enforced, not warned** — Claude Code's `/goal` has a 4,000-character ceiling. goaljaby applies a 5-stage compact (normalize → externalize → abbreviate → summarize milestones → trim irrelevant boilerplate) and aborts cleanly with a structural overflow report if it still can't fit. No silent truncation, no advisory warnings.
- **PROTECTED_CLAUSES are uncuttable** — Stop condition, scope lock, 3-attempt rule, doc-read directive, and PROGRESS update — all five are required and verified by regex in Step 7. If any clause is missing post-compact, the output is discarded.
- **Both CLIs in one skill** — Claude Code (v2.1.139+, hooks-based, no pause/resume) and Codex CLI (0.128.0+, opt-in, pause/resume supported). goaljaby branches all 5 doc generations + the final handoff on `target_cli` chosen in Step 2.

## How it works

```
PRD directory
     |
     v
[Step 0-1] Pre-check + analysis
     |   - No PRD? Delegate to /show-me-the-prd
     |   - Extract acceptance / non-goals / task type
     v
[Step 2-4] Interview (1-2 rounds)
     |   target_cli, task_type, validation, strictness, milestones
     v
[Step 5] Slot-fill into 5 documents
     |   VALIDATION.md / RECOVERY.md / PLAN.md / PROGRESS.md / goal-command.md
     |   (+ codex-handoff.md if target_cli != claude_code)
     v
[Step 6] Auto-compact goal-command.md to ≤4,000 chars
     |   normalize → externalize → abbreviate → summarize → trim
     v
[Step 7] Deterministic self-verification
     |   grep -P against 5 PROTECTED_CLAUSES + char count
     |   On failure → structural overflow report + DISCARD output
     v
[Step 8-9] Show outputs + REQUIRED human approval gate
     v
[Step 10] Paste-back guide (Claude Code) OR handoff prompt (Codex)
```

## Outputs

```
[PRD directory]/
├── VALIDATION.md      ← acceptance mapping + Not Done If
├── RECOVERY.md        ← 3-attempt rule + scope lock + task-type recovery
├── PLAN.md            ← milestones (≤5) + Final Completion Criteria
├── PROGRESS.md        ← empty initial template
├── goal-command.md    ← /goal body (always ≤4,000 chars)
└── codex-handoff.md   ← (Codex target only) first-message template for a new Codex session
```

Each document's emphasis depends on the task type (Feature / Bugfix / UI / Doc / Migration / Eval).

---

## Task types

| Task type | VALIDATION emphasis | RECOVERY emphasis | /goal template |
|-----------|---------------------|-------------------|----------------|
| Feature | Unit + integration tests | Scope lock | F-2 |
| Bugfix | Original repro + regression | No unrelated module edits | F-1 |
| UI | Screenshots + viewport checks | Design token protection | F-3 |
| Doc | Section-by-section review | Frozen-once-approved sections | F-4 |
| Migration | Parity check + rollback | Public API immutability | F-5 |
| Eval | Score vs baseline | One prompt change at a time | F-6 |

Task type is auto-estimated from PRD content (weighted Korean + English keyword matching) and finalized by the user in Step 3.

---

## Core promises

- **`goal-command.md` is always ≤4,000 characters** — If compact can't fit, the file is not saved; a structural overflow report is printed instead. No warnings-and-shrug.
- **PROTECTED_CLAUSES are inviolate** — Stop condition, scope lock, 3-attempt rule, doc references, and PROGRESS update are protected by regex at compact time.
- **CLI branch decides everything** — `target_cli` chosen in Step 2 shapes RECOVERY phrasing, goal-command body, and Step 10 handoff form.
- **Step 9 approval gate cannot be bypassed** — Even when auto-execution is wired, Step 10 will not run until human approval comes back from Step 9.
- **Self-demonstrating** — building this skill itself is a valid goaljaby target.

---

## Requirements

- **Claude Code v2.1.139+** (or Codex CLI 0.128.0+)
- Claude Code: hooks must be active (`disableAllHooks` / `allowManagedHooksOnly` unset)
- Codex CLI: enable goals via `~/.codex/config.toml` `[features] goals = true` or `/experimental`
- *(Optional)* `/show-me-the-prd` plugin — goaljaby delegates to it in Step 0 when no PRD exists

CLI differences in detail: `skills/goaljaby/references/cli-rules.md`.
