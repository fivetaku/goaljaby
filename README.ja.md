# goaljaby (골잡이)

<p align="center">
  <img src="assets/goaljaby-hero-01.png" alt="goaljaby" width="320">
</p>

> **Claude Code 向けの PRD→/goal ブリッジ — 韓国語のレビュー文書を確認して承認すれば、すぐにゴール作業が始まる。**

goaljaby は PRD フォルダ（手動作成でも `/show-me-the-prd` の成果物でも）を受け取り、検証／リカバリのループを包む**韓国語レビュー文書 5 種** — VALIDATION、RECOVERY、PLAN、PROGRESS、そして `/goal` コマンド本文 — を自動生成します。韓国語のレビュー要約は（余分なファイルを作らず）チャットに直接表示され、ハンドオフ用に PROGRESS.md の先頭へ 4 行サマリーが前置されます。韓国語で読み、一度承認すれば、次のターンでゴールが始まります — `/goal` の行はアシスタントが代わりに出力します。

### 4. 実行

既存の PRD ディレクトリがある場合 — **絶対パス推奨**：

```
/goaljaby /Users/<username>/my-project/PRD/
```

PRD がない場合 — goaljaby が `/show-me-the-prd` への委譲を提案します：

```
/goaljaby
```

- "골잡이 호출"
- "골 셋업 만들어줘"
- "PRD에서 골 만들어줘"
- "set up goal from PRD"

## なぜ goaljaby なのか

- **PRD だけでは足りない** — PRD は*何を*作るかを定義します。`/goal` には*完了をどう証明するか*と*間違った方向に進んだときどう立て直すか*まで必要です。どちらかが欠けると、ゴールは「それらしく見える結果」で止まるか、スコープ外へ流れていきます。
- **英語のボイラープレートではなく、韓国語のレビュー文書** — 生成される文書は韓国語ファーストなので、ユーザーは承認前に本当に読んでレビューできます。見出しは 필수 검증、완료 기준 매핑、완료로 보지 않는 조건 — 英語の対応語ではありません。
- **レビュー要約はチャットに直接** — 余分な brief ファイルは作りません。Step 8 で韓国語のレビュー要約をチャットにそのまま表示し、ハンドオフが機能するよう PROGRESS.md の先頭へ 4 行サマリーを前置します。
- **一度承認すれば作業が始まる** — 承認後、アシスタントが応答の最終行に `/goal {本文}` を出力し、次のターンでセッションがゴールを開始します。韓国語のレビュー要約を読んで承認するだけで、作業が進みます。
- **4,000 字コンパクトは警告ではなく強制** — Claude Code の `/goal` には 4,000 字の上限があります。goaljaby は 5 段階のコンパクトを適用し、それでも収まらなければ構造的オーバーフロー報告を出してクリーンに中断します。黙った切り捨てはありません。
- **PROTECTED_CLAUSES は削れない** — 停止条件、スコープロック、3 回ルール、文書参照指示、PROGRESS 更新は、コンパクト後に韓国語+英語 OR の正規表現で検証されます。1 つでも欠けていれば出力は破棄されます。
- **人間の承認ゲートは必須** — Step 9 の AskUserQuestion はバイパスできません。ゴールは明示的な承認の後にだけ始まります。

## 仕組み

```
PRD ディレクトリ
     │
     ▼
[Step 0-1] 事前チェック + 分析
     │   PRD がない？ → /show-me-the-prd に委譲
     │   acceptance / non-goals / 作業タイプを抽出
     ▼
[Step 2-4] インタビュー（1〜2 回）
     │   task_type、検証方法、厳格度、マイルストーン
     ▼
[Step 5] 韓国語文書 5 種のスロット充填
     │   VALIDATION / RECOVERY / PLAN / PROGRESS / goal-command
     ▼
[Step 6] goal-command.md を 4,000 字以内へ自動コンパクト
     │   正規化 → 外部化 → 省略化 → 要約 → 切り詰め
     ▼
[Step 7] 決定論的な自己検証
     │   韓国語+英語 OR の PROTECTED_CLAUSES 5 種を grep -P で照合
     │   + 文字数チェック + 英語見出し残留チェック
     │   失敗時 → 構造的オーバーフロー報告 + 破棄
     ▼
[Step 8] 韓国語レビュー要約をチャットに表示
     │   + PROGRESS.md の先頭に 4 行サマリーを前置（ハンドオフ用）
     ▼
[Step 9] AskUserQuestion — 承認 / 修正 / 後で / キャンセル
     ▼
[Step 10] 承認時：
          • PROGRESS.md に開始時刻を記録
          • 応答の最終行として `/goal {本文}` を出力
          • 次のターンでセッションがゴールを開始
```

## 生成ファイル

```
[PRD directory]/
├── VALIDATION.md      ← 필수 검증 / 완료 기준 매핑 / 완료로 보지 않는 조건
├── RECOVERY.md        ← 기본 원칙 / 실패 루프 / 재시도 한계 / scope 잠금
├── PLAN.md            ← 목표 / 마일스톤(≤5) / 최종 완료 기준
├── PROGRESS.md        ← 빈 초기 템플릿 + Step 8 4-line summary prepended
└── goal-command.md    ← /goal 본문 (한국어, ≤4,000 chars)
```

5 種すべてが韓国語ファーストです。Step 8 のレビュー要約はチャットにのみ表示され、別ファイルは作られません。ファイル名、コマンド識別子、シェルコマンドはそのままです。

## 作業タイプ

| 作業タイプ | VALIDATION の重点 | RECOVERY の重点 | /goal テンプレート |
|-----------|---------------------|-------------------|----------------|
| 기능 구현（機能実装） | ユニット + 統合テスト | スコープロック | F-2 |
| 버그 수정（バグ修正） | 元の再現 + 回帰テスト | 無関係モジュールの変更禁止 | F-1 |
| UI 구현（UI 実装） | スクリーンショット + ビューポート検証 | デザイントークン保護 | F-3 |
| 문서 집필（ドキュメント執筆） | セクションごとのレビュー | 承認済みセクションの凍結 | F-4 |
| 마이그레이션（マイグレーション） | パリティチェック + ロールバック | public API 不変 | F-5 |
| eval 개선（eval 改善） | ベースライン比のスコア | プロンプト変更は一度に 1 つ | F-6 |

作業タイプは PRD の内容から自動推定され（韓国語+英語キーワードの重み付きマッチング）、Step 3 でユーザーが最終確定します。

## コアの約束

- **生成文書は韓国語ファースト** — Step 7 で英語見出しの残留を検出します。見つかれば結果は破棄。半分英語の出力は出ません。
- **レビュー要約はチャットだけ** — 別の brief ファイルは作りません。PROGRESS.md の先頭に 4 行サマリーを残してハンドオフだけを担保します。
- **`goal-command.md` は常に 4,000 字以下** — コンパクトで収まらなければファイルは保存されず、代わりに構造的オーバーフロー報告が表示されます。
- **PROTECTED_CLAUSES は不可侵** — 停止条件、スコープロック、3 回ルール、文書参照、PROGRESS 更新は韓国語+英語 OR の正規表現で守られます。
- **Step 9 の承認ゲートはバイパス不可** — Step 10 は明示的な人間の承認の後にだけ発火します。
- **自己実証** — このスキル自体を作る作業も、goaljaby の正当なターゲットになります。
